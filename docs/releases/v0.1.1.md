# Heft v0.1.1

`v0.1.1` is a follow-up prerelease that tightens the public contract after the
initial `v0.1.0` release preparation.

## Highlights

- fixes stale `ledger_last_error()` behavior on several report/classification ABI paths
- makes the local and CI quality gate enforce formatting instead of rewriting it
- clarifies nullable C ABI parameters in `heft.h`
- moves large benchmark buffers off the stack
- documents bounded temporary map usage more explicitly in a few hot paths

## Positioning

Heft remains:

- an embedded accounting engine
- SQLite-backed
- Zig-first with a C ABI
- focused on transactional correctness, accounting lifecycle, and deterministic behavior

## Stability

This is still a `0.x` prerelease line.

- the accounting core is strong and increasingly well-covered
- public API shape may still evolve before `1.0`
- release packaging, CI, and benchmarks are now cleaner than in `v0.1.0`
