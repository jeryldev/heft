# Heft v0.1.0

`v0.1.0` is the first public release candidate for Heft as an embedded
accounting engine.

## Positioning

Heft is:

- an embedded accounting engine
- SQLite-backed
- Zig-first with a C ABI
- focused on transactional correctness, accounting lifecycle, and deterministic behavior

Heft is not:

- a network service
- a distributed ledger
- a complete end-user accounting application

## Highlights

- designation-based accounting policy per book of account
- period-aware posting, close, reopening, and opening carry-forward
- strong accounting invariants around reversal, cache correctness, and open items
- reporting, comparative reporting, classifications, dimensions, budgets, and exports
- broad in-memory and ABI integration coverage
- benchmark harness with setup-vs-operation separation and feature coverage across:
  - reporting
  - comparative reports
  - exports
  - budgets
  - open items
  - classifications
  - dimensions
  - key C ABI buffer endpoints

## Stability

This is still a `0.x` release.

- the engine is already strong on correctness and lifecycle behavior
- the public API may still change before `1.0`
- packaging, CI, and examples are improving but not final

See [Stability Statement](../stability.md).

## Suggested release message

Heft `0.1.0` is ready to be evaluated as an embedded accounting core for
applications that want a deterministic, SQLite-backed ledger engine with Zig
and C integration points.
